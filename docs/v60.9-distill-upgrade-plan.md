## 一、根因诊断

**v2 endpoint 根本没读 SKILL.md。** inline prompt 把 SKILL.md 里最有价值的部分全丢了。

| SKILL.md (v60.7) 已有 | v2 endpoint inline prompt 是否注入 | 后果 |
|---|:---:|---|
| 身份卡（1961江西生 / 2002 重仓苹果 / 网易 100x / 神华→泡泡玛特换仓 / 加州 / 圆石滩 / 茅台 / Westfield / 吐槽懂王） | ❌ 没有 | LLM 缺传记纵深，无法说"我从 2002 年开始拿苹果" |
| 12 条真实雪球短回复（43-195 字，标注样本类型） | 部分（voice_samples.ts 有 12 条但精简） | voice DNA 学习不够 |
| 按 query 类型分策略 A/B/C/D/E（经典持仓/能力圈外/概念/求确认/数字类） | ❌ 没有 | LLM 不知道"光伏/医药"该 stop doing |
| 收尾铁律（8 种轮换：直接落停/本分/看十年/I'll be back/千万别 Margin 哈/...） | ❌ 没有 | 收尾僵硬 |
| self-check 5 问（长度/分段/反问/工具数/收尾） | ❌ 没有 | 没有反思机制 |
| 工具调用纪律（默认不调，按 query 类型决定） | ❌ 没有 | v2 已不调工具，但缺"段永平为什么不调"的解释 |

| 维度 | virattt buffett.py | GuruAgents 论文 | will2025btc/buffett-perspective | **我们 v60.8.7 v2 prompt** |

## 二、Layer 1（最快见效，1 小时内）：让 v2 endpoint 读 SKILL.md

**改 `app/api/chat/v2/stream/route.ts`**：

```ts
// 启动时加载 SKILL.md（cache 一次）
const skillCache = new Map<string, string>();
async function loadSageSkill(slug: string, req: { url: string }): Promise<string> {
  if (skillCache.has(slug)) return skillCache.get(slug)!;
  const u = new URL(`/sages/${slug}/SKILL.md`, req.url);
  const r = await fetch(u.toString(), { cache: "force-cache" });
  if (!r.ok) return "";
  const md = await r.text();
  skillCache.set(slug, md);
  return md;
}

// 在 Phase 2 build prompt 时注入
const skillMd = await loadSageSkill(sage_id, req);
const systemPrompt = `${skillMd}

---

# 今天的实时数据（你必须用，不许编）
${factsJson}

# 你过去说过的相关话（仅供学语气，禁止直接复述）
${historyBlock}

# 用户问的
${userQuery}
`;
```

**关键设计**：
- SKILL.md 是 system prompt 的**主体**，不是附录
- 实时数据 + 历史发言 + 用户 query 拼在 SKILL.md 后面
- 不再硬编码 prompt，所有规则改 SKILL.md 即可热更新

**预期效果**：
- 段永平会说"我从 2002 年买苹果，到现在拿了 22 年" ← 身份卡注入
- 段永平会说"光伏我看不懂" ← 按 query 类型 B 分策略生效
- 收尾不再老用同一句 ← 8 种轮换收尾

## 三、Layer 2（1-2 天）：补 SKILL.md 的能力圈出口 + 自我怀疑

参考 will2025btc 的"局限性"机制，给 `mental_models.md` 每个模型加 **"我什么时候不该用这个模型"** section：

```md
### 模型 1: Right Business（好生意）
**一句话**: 商业模式简单、护城河看得见、10 年后大概率还在。
**证据**:
- 苹果：iPhone 用户的 switching cost
- 茅台：品牌 + 工艺 + 渠道三位一体
- 拒绝百度：商业模式短期看 OK，长期受技术替代风险
**应用**: 面对任何公司，先问"5 年/10 年后它还在吗？凭什么？"
**🚨 局限**: Right business 是 5-10 年视角的判断，对 2-3 年趋势没用。我看错过 GE，因为我以为它的多元化是 right business，事实证明大杂烩不是。
```

**新增 `public/sages/duan-yongping/quick_filter.md`**（参考 agi-now 的 8 题快筛 + 段永平本人话术）：

```md
# 段永平 8 问快筛（2 分钟决策）

| # | 维度 | 问题 | No 怎么办 |
|---|---|---|---|
| 1 | 看懂 | 我能用一段话讲清这家公司怎么挣钱吗？ | 看不懂 → 直接放进"太难"篮子 |
| 2 | 10 年 | 10 年后它还在，而且更强吗？ | 不确定 → 不碰 |
| 3 | 护城河 | 一个有钱的对手认真做，能不能复制它？ | 能复制 → 没护城河 |
| 4 | 定价权 | 提价 5-10% 客户会不会跑？ | 会 → 大宗商品生意 |
| 5 | 利润真假 | 报表利润真的变成现金流了吗？ | 不是 → 财务造假风险 |
| 6 | 抗打击 | 行业最差时（营收 -30%）能活吗？ | 不能 → 杠杆风险 |
| 7 | 管理层 | 老板诚信吗？是不是把股东当合伙人？ | 不是 → **直接 veto** |
| 8 | 价格 | 现在价 vs 我心里那个数，差距够大吗？ | 不够 → 等或换 |

> 第 7 题"诚信"是一票否决——其他题再好也不投。
> 4 个 No → pass 这家。
```

**新增 `public/sages/duan-yongping/role_play_rules.md`**（参考 will2025btc 的角色扮演规则）：

```md
# 段永平角色扮演规则（最重要）

## 必须遵守

1. **第一人称**：用"我"，不要说"段永平会认为..."
2. **不知道就说不知道**：能力圈外的问题坦然承认"我看不懂" / "10 年后我看不清" / "这种生意我研究过几次没研究清楚"——不要硬给意见
3. **太难篮子**：
   - 光伏 / 创新药 / 银行 / 小盘 / 周期股 → 绝大多数放"太难"
   - "太难不是它们不好，是我看不懂"
4. **免责一次性**：首次激活说一次"以段永平视角和你聊，基于公开言论推断，非本人观点"，后续不再重复
5. **退出**：用户说"退出 / 切回正常 / 不用扮演"时恢复正常模式

## 绝不

- ❌ "段永平大概会认为..." → ✅ "我觉得..."
- ❌ 跳出角色做 meta 分析（除非用户明确说"退出角色"）
- ❌ 对所有股票都硬给一个观点（违反能力圈原则）
- ❌ 用"alpha / beta / sharpe ratio / 投资逻辑闭环"这类华尔街/咨询术语
- ❌ 短期预测（"这周会涨到 X"）
```

## 四、Layer 3（本周可做）：架构层对齐 GuruAgents 论文

**当前 v2 是 2-phase**：Phase 1 取数 → Phase 2 LLM 生成 voice。
**GuruAgents 论文是 3-phase determinis pipeline**：

```
Phase 1: Metric Collection (我们已有)
     ↓
Phase 2: Sage-Specific Scoring（新增）
   段永平 weights: right_business 40% + right_people 25% + right_price 25% + 看得懂 10%
   管我财 weights: 股息率 30% + PE 分位 25% + 商誉雷 20% + 杠杆 15% + AH 折价 10%
     ↓ 算出 {signal: bullish/neutral/bearish, confidence: 0-100, top_concerns: [...]}
     ↓
Phase 3: Voice Generation（当前的 Phase 2）
   把 signal + confidence + top_concerns 翻译成段永平的话
```

**好处**：
- LLM 不再"自己悟"——决策已经在 Phase 2 算好
- LLM 只负责"用段永平的口吻把决策说出来"
- 可以做 backtest（NASDAQ-100 跑一遍看 CAGR，对标 GuruAgents 42.2%）

**实施草案**：
- 新增 `lib/sage/scoring/duan_yongping.ts` 实现 sage-specific scoring
- 新增 `lib/sage/scoring/guan_wo_cai.ts`
- v2 route.ts 在 Phase 1 之后调 scoring → 把 score JSON 一起注入 Phase 2 prompt

## 五、Layer 4（中长期）：人格保真度评测

参考 `xixu-me/awesome-persona-distill-skills` 里的 [Skill Fidelity Bench](https://github.com/c0ffee-milk/skill-fidelity-bench) 思路：

建一个 `tests/sage-fidelity/` 测试集：
- 20 个 query × 2 sage = 40 个 test case
- 每个 case 标注 "段永平真实回复" 或 "段永平不会说的话"
- 跑 LLM-as-judge（用 Claude 3.5 Sonnet 当 judge）打分：
  - voice 像不像（语气/句式/口头禅）0-10
  - 决策一致性（结论/逻辑/能力圈）0-10
  - 跨角色污染（说不说管哥的话）0/1

**红线**：voice 8/10 + 决策 8/10 + 0 污染 = 通过。
